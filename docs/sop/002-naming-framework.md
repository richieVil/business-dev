# SOP: Universal Naming Framework (002-naming-framework)

## 1. Objective & Scope
The objective of this SOP is to generate a brand name that functions as a "Neural Shortcut"—easy to remember, hard to forget, and strategically aligned with the "Vibe" of the solution. It ensures the name is technically ownable and free of legal collisions.

---

## 2. Prerequisites (Input Requirements)
Naming begins **ONLY** after the "Strategic Source" is established:
1.  **VMC (Value Mapping Canvas):** Identification of the "Mechanism."
2.  **VPC (Value Proposition Canvas):** Identification of the "Emotional Hook."
3.  **Tech Architecture:** Identification of the platform nature.

---

## 3. Critical to Quality (CTQ) Metrics
*   **Cognitive Fluency:** 6–8 characters, 1–2 syllables.
*   **Zero-Explanation Spelling:** Intuitive spelling after one auditory exposure.
*   **Linguistic Safety:** No negative connotations in target markets.
*   **Strategic Elasticity:** Must support pivot/expansion (e.g., not "Factura-Only").
*   **Research Integrity:** All "Collisions" (failures) must be documented to prevent re-research.

---

## 4. The 6-Phase Process

### Phase 1: Semantic Extraction
1.  **Action Verbs:** List 10 verbs describing the technical "mechanism."
2.  **Emotional Gains:** List 10 nouns/adjectives describing the user's state.
3.  **The Masterbrand Narrative:** Write a 3-sentence "vibe" description.

### Phase 2: High-Volume Ideation
Generate a pool of **20+ candidates** across four archetypes:
1.  **Utility (Abstract):** Short, efficiency-focused (e.g., Xero).
2.  **Authority (Root-based):** Latin/Greek roots (e.g., Veris).
3.  **Metaphor (Object):** Symbolic nouns (e.g., Anchor).
4.  **Compound/Modern:** Two-word blends (e.g., Flexport).

### Phase 3: Phonetic & Visual Engineering
Filter the pool for sound-symbolism:
*   **Hard Plosives (K, T, P, B, D, G):** For Speed/Precision.
*   **Soft Fricatives (F, S, V, Z, L, M):** For Trust/Security.
*   **Barcode Check:** Ensure the lowercase name doesn't look like a barcode (e.g., "liilil").

### Phase 4: Friction Filtering
Stress test the top survivors:
1.  **The Shout Test:** Speak it in a noisy room. If the listener asks "What?", it fails.
2.  **The Spelling Test:** Ask a 3rd party to spell it. Any deviation fails.

### Phase 5: The Forensic Search Protocol
Execute the search templates for the finalists. **MANDATORY:** Every candidate that is eliminated must be recorded in the **Elimination Audit Trail** with a specific failure code:
*   **[PHON]:** Phonetic failure (Shout/Spelling test).
*   **[TM]:** Trademark/Legal collision (Incumbents).
*   **[DOM]:** Domain/Social handle unavailable or premium-priced.
*   **[NAR]:** Narrative mismatch (Does not evoke the Masterbrand Narrative).
*   **[STRAT]:** Strategic Elasticity failure (Too narrow or restricts growth).

### Phase 6: Final Selection (Conviction Matrix)
Rate the survivors using the **Integer-5 scale** on Memorability, Scalability, Vibe-Fit, and Ownability.

---

## 5. The Naming Dossier Output (`05-identity-branding.md`)
The naming exercise must produce a document with these sections:
1.  **Semantic Extraction:** Action verbs, Emotional gains, and Narrative.
2.  **The Ideation Pool:** The full list of 20+ initial candidates.
3.  **Elimination Audit Trail:** A table documenting the death of every candidate (Name | Failure Code | Evidence/Logic).
4.  **Marketing & Brand Strategy:**
    *   **Tagline Generation:** 3 options for a 3-5 word secondary hook.
    *   **Visual Identity Direction:** Core color palette (Hex codes) and Typography vibe.
    *   **Domain Strategy:** Primary vs. Secondary (e.g., `[name].mx` vs `get[name].mx`).
    *   **Brand Hierarchy:** How sub-products are named (e.g., `[Name] Vault`, `[Name] Pulse`).
5.  **The Conviction Matrix:** Final scoring and Masterbrand selection.

---

## 6. Definition of Done (DoD)
- [ ] A Masterbrand is selected via the Conviction Matrix.
- [ ] The "Collision Report" documents all research failures.
- [ ] A "Brand Soul" paragraph is written explaining the choice.
- [ ] Name initialized as the project codename in the directory and TaskBoard.
