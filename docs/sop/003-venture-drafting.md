# SOP: Venture Drafting Standards (003-venture-drafting)

## 1. Objective
To standardize the visual and structural representation of strategic canvases. This ensures every "Incubee" spec is auditable, professional, and consistent with the Venture Architect methodology.

---

## 2. The Value Proposition Canvas (VPC)
The VPC must be rendered as a comparative analysis between the **Customer Profile** (External Reality) and the **Value Map** (Internal Solution).

### 2.1 The Profile Table
| Category | Definition | Factura-Flow Example |
| :--- | :--- | :--- |
| **Economic Buyer** | Who pays/authorizes. | SMB Owner / CFO. |
| **End User** | Who operates the tool. | Cashier / Receptionist. |
| **Jobs (JTBD)** | Functional/Legal tasks. | Issue CFDI 4.0 / Map SAT Keys. |
| **Pains** | Bottlenecks/Risks. | "The Line" at checkout / SAT downtime. |
| **Gains** | Desired outcomes. | One-click deductions / Automated vault. |

---

## 3. The Value Mapping Canvas (VMC)
The VMC must define the **"Mechanism"** using the **TOF Model** (Trigger, Orchestrator, Fulfillment).

### 3.1 The TOF Table
| Layer | Component | Description |
| :--- | :--- | :--- |
| **Trigger** | Ingestion | How data enters (e.g., Webhook, OCR, Sensor). |
| **Orchestrator** | Transformation | The "Black Box" logic (e.g., SKU-to-SAT Mapping). |
| **Fulfillment** | Delivery | The Value Handshake (e.g., WhatsApp PDF delivery). |

### 3.2 The Handshake (Step-by-Step)
A numbered list documenting the user-to-system interaction loop. No step should be missing.
1. [Action] -> 2. [System Response] -> 3. [Output].

## 4. The Business Model Canvas (BMC)
The BMC defines the **Strategic Ecosystem**. To prevent "skeleton" summaries, it must be drafted as a deep-dive into the "Who" and the "Where," using the following mandatory subsections:

### 4.1 Customer Segments (The Personas)
Must define at least three distinct personas using the following schema:
*   **The Persona:** Professional title and business type (e.g., "The Modern Retailer").
*   **Key Technology:** The specific software they already use (e.g., Square, Loyverse).
*   **The "Micro-Moment" of Pain:** The exact 60-second operational bottleneck.
*   **The Economic Motivation:** Why they will sign the check (e.g., "Operational Speed" vs. "Regulatory Fear").

### 4.2 Value Propositions (The Stakeholder Matrix)
Must provide a matrix mapping the specific "Magic" to every party in the loop:
*   **The Seller (Economic Buyer):** e.g., "Reduce checkout bottlenecks by 90%."
*   **The Buyer (The Guest):** e.g., "One-click deductions across the entire network."
*   **The Staff (The User):** e.g., "Removing SAT portal anxiety from the daily workflow."
*   **The Influencer (The Accountant):** e.g., "Zero-labor month-end reconciliation."

### 4.3 Channels (The Distribution Flywheel)
Must document the "Parasitic" growth path:
*   **The Platform Parasite:** Which App Store/Marketplace are we listing in?
*   **The Transactional Distribution:** How does the "Receipt" or "Handshake" sell the next user?
*   **The Influencer Loop:** How does the "Free for Gatekeeper" model drive B2B2B adoption?

### 4.4 Customer Relationships (The Stickiness)
*   **The Invisible Assistant:** Describe the zero-friction onboarding within the communication channel.
*   **The Identity Moat:** How the "Vault" makes it harder for a user to switch to a competitor.
*   **Contextual Branding:** How the bot maintains the Seller's identity (Logos, Headers) while using a central number.

### 4.5 Key Activities & Resources
*   **Activities:** Must include "Orchestration Management," "Vault Security," and "Catalog Mapping Sprints."
*   **Resources:** Must include the "Identity Ledger (IP)," "CSD Vault," and "WABA Infrastructure."

### 4.6 Key Partnerships
*   **The Legal Backbone:** Identify the PACs or regulatory partners.
*   **The Reseller Network:** Identify the POS distributors or local "influencers" who get a cut.
*   **The Gatekeepers:** Strategic alliance with accounting firms.

### 4.7 Revenue Streams (The Pricing Engine)
*   **Pricing Tiers:** Must explicitly state the "Base + Usage" tiers (e.g., Tier 1, Tier 2, Enterprise).
*   **Success Fee Logic:** If applicable, define the % take-rate on recovered funds or facilitated loans.
*   **Policy:** Must state the "No-Unlimited" rule if variable costs are significant.

### 4.8 Cost Structure (The Unit Floor)
*   **Variable Costs:** List all per-transaction fees (WhatsApp, PAC, AI tokens).
*   **Fixed Costs:** Serverless infra, WABA maintenance, security auditing.
*   **Unit Economics:** Document the total "Cost Floor" per successful unit.

## 5. The Venture Dossier Structure (The 5-File Standard)
To prevent "skeleton" drafts and ensure total technical and financial visibility, every venture must be documented across five distinct files:

### 5.1 File 1: `01-strategy-vpc.md` (The Pain)
*   **Content:** Mission, OSV, BOI Matrix (Phase 2), and the VPC Profile/Map tables.
*   **CTQ:** Must identify the "Economic Buyer" and their "Micro-Moment of Pain."

### 5.2 File 2: `02-mechanism-vmc.md` (The Tech)
*   **Content:** The TOF Table (Trigger, Orchestrator, Fulfillment) and the step-by-step Handshake logic.
*   **CTQ:** Must define the "Orchestrator" (The Black Box logic) and the "Guest UX" for non-paying parties.

### 5.3 File 3: `03-business-bmc.md` (The Strategy)
*   **Content:** Persona Deep-Dives, Channel Flywheel, Stakeholder Matrix, Partnerships, and the Revenue/Cost model.
*   **CTQ:** Must define the "Influencer-as-Distribution" logic and the "Variable Cost Floor" vs. Price.

### 5.4 File 4: `04-meta-analysis.md` (The Audit)
*   **Content:** ROIE/CAL Deep-Dive, Strategic Risk Ledger, and Pivot Scenarios.
*   **CTQ:** Must perform the final "Conviction Check"—analyzing if the BMC defined in File 03 is robust enough to move to implementation.

### 5.5 File 5: `05-identity-branding.md` (The Brand)
*   **Content:** The full 20+ name pool, the forensic trademark search results, and the selection of the Masterbrand via the Conviction Matrix.
*   **CTQ:** Must document the "Shout Test" and "Spelling Test" for the winner.
