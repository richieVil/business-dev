# Venture Architect: High-Fidelity (VA-HF)

## 1. Overview
The **High-Fidelity (VA-HF)** system is the second-generation strategic engine for `business-dev`. While the legacy VAP focuses on high-volume filtering, VA-HF focuses on high-resolution engineering. It is designed to ensure that no venture moves to code without a validated technical mechanism, a distribution flywheel, and protected unit economics.

---

## 2. Core Differences (Legacy vs. VA-HF)

| Aspect | Legacy (VAP) | Modern (VA-HF) |
| :--- | :--- | :--- |
| **Primary Goal** | **Triage:** Filter "Good" from "Bad." | **Engineering:** Build the "How." |
| **Mathematical Logic** | Geometric Mean (Single "1" kills). | Weighted Sum (Prioritizes "Stakes"). |
| **Metric Focus** | TAM & Feasibility. | **Mechanism & Leverage.** |
| **Growth Strategy** | Direct Sales / Marketing. | **CAL (Platform Parasitism).** |
| **Moat Definition** | Static IP / Barrier. | **Data Gravity / Identity Vault.** |
| **Documentation** | Single descriptive card. | **4-File Strategic Dossier.** |

## 3. The VA-HF Methodology (SOP 001-003)

### Phase 1: The Seed
Establishing the **One-Sentence Value (OSV)** and the functional **Seed Slug** to prevent premature branding.

### Phase 2: Strategic Triage (BOI)
Using weighted math to audit the "Force" behind the idea. 
*   *Target:* > 4.25 (Weighted 1-5 scale).

### Phase 3: Mechanism Design (TOF)
Deconstructing the product into its technical reality:
*   **Trigger:** How data enters.
*   **Orchestrator:** The "Black Box" logic.
*   **Fulfillment:** The value delivery.

### Phase 4: Execution Economics (ROIE & CAL)
Ensuring the business is "Easy Money" by measuring **Return on Implementation Effort (ROIE)** and finding **Customer Acquisition Leverage (CAL)**.

### Phase 5: Platform & Identity
Designing for scale by choosing between a standalone tool and a **Centralized Identity Network**.

## 4. Documentation Standards
Every VA-HF venture is stored in `ventures/[seed-slug]/` and must contain:
1.  **`01-strategy-vpc.md`**: The Pain/Gain fit and the "Micro-Moment of Pain."
2.  **`02-mechanism-vmc.md`**: The TOF table and step-by-step Handshake.
3.  **`03-business-bmc.md`**: Persona deep-dives, Channel Flywheels, and Partnerships.
4.  **`04-meta-analysis.md`**: Unit Cost Simulations, Risk Ledgers, and Pivot Scenarios.
